Django settings control your project's configuration. Settings are defined in your project's `settings.py` file.

## Core Settings

### DEBUG

```python
DEBUG = False
```

Enables debug mode. When `True`, Django displays detailed error pages with stack traces.

<Note>
Never deploy a site with `DEBUG = True` in production. It exposes sensitive information and impacts performance.
</Note>

### SECRET_KEY

```python
SECRET_KEY = ""
```

A secret key used for cryptographic signing. Must be unique and kept secret.

<ParamField path="SECRET_KEY" type="string" required>
  A unique, unpredictable value used for security-related operations.
</ParamField>

<ParamField path="SECRET_KEY_FALLBACKS" type="list">
  List of secret keys used to verify signatures during key rotation.
</ParamField>

### ALLOWED_HOSTS

```python
ALLOWED_HOSTS = []
```

List of host/domain names that Django can serve. Required when `DEBUG = False`.

<ParamField path="ALLOWED_HOSTS" type="list" required>
  List of strings representing host/domain names. Use `"*"` to match anything, or `".example.com"` for subdomains.
</ParamField>

## Database Settings

### DATABASES

```python
DATABASES = {}
```

Database connection configuration. If left empty, defaults to the dummy backend.

<ParamField path="DATABASES" type="dict" required>
  Dictionary mapping database aliases to connection settings.
</ParamField>

```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'mydatabase',
        'USER': 'mydatabaseuser',
        'PASSWORD': 'mypassword',
        'HOST': '127.0.0.1',
        'PORT': '5432',
    }
}
```

<ParamField path="DATABASE_ROUTERS" type="list">
  List of classes used to implement database routing behavior.
</ParamField>

<ParamField path="DEFAULT_AUTO_FIELD" type="string">
  Default primary key field type.
  
  Default: `"django.db.models.BigAutoField"`
</ParamField>

## Application Settings

### INSTALLED_APPS

```python
INSTALLED_APPS = []
```

List of strings representing installed Django applications.

<ParamField path="INSTALLED_APPS" type="list" required>
  List of application names or `AppConfig` paths.
</ParamField>

```python
INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'myapp',
]
```

### MIDDLEWARE

```python
MIDDLEWARE = []
```

List of middleware classes to use. Order is important: applied in order during request phase, reverse order during response phase.

<ParamField path="MIDDLEWARE" type="list">
  List of middleware class paths.
</ParamField>

```python
MIDDLEWARE = [
    'django.middleware.security.SecurityMiddleware',
    'django.contrib.sessions.middleware.SessionMiddleware',
    'django.middleware.common.CommonMiddleware',
    'django.middleware.csrf.CsrfViewMiddleware',
    'django.contrib.auth.middleware.AuthenticationMiddleware',
    'django.contrib.messages.middleware.MessageMiddleware',
    'django.middleware.clickjacking.XFrameOptionsMiddleware',
]
```

## Internationalization & Localization

<ParamField path="LANGUAGE_CODE" type="string">
  Language code for this installation.
  
  Default: `"en-us"`
</ParamField>

<ParamField path="TIME_ZONE" type="string">
  Time zone for this installation. When `USE_TZ = True`, this is the default user time zone.
  
  Default: `"America/Chicago"`
</ParamField>

<ParamField path="USE_TZ" type="bool">
  Enable timezone-aware datetimes.
  
  Default: `True`
</ParamField>

<ParamField path="USE_I18N" type="bool">
  Enable internationalization machinery.
  
  Default: `True`
</ParamField>

<ParamField path="LOCALE_PATHS" type="list">
  List of directories to search for translation files.
  
  Default: `[]`
</ParamField>

## Static Files

<ParamField path="STATIC_URL" type="string">
  URL that handles static files served from `STATIC_ROOT`.
  
  Example: `"/static/"` or `"http://static.example.com/"`
</ParamField>

<ParamField path="STATIC_ROOT" type="string">
  Absolute path to the directory where static files are collected.
  
  Example: `"/var/www/example.com/static/"`
</ParamField>

<ParamField path="STATICFILES_DIRS" type="list">
  List of additional locations for static files.
  
  Default: `[]`
</ParamField>

<ParamField path="STATICFILES_FINDERS" type="list">
  List of finder classes that know how to find static files.
  
  Default:
  ```python
  [
      "django.contrib.staticfiles.finders.FileSystemFinder",
      "django.contrib.staticfiles.finders.AppDirectoriesFinder",
  ]
  ```
</ParamField>

## Media Files

<ParamField path="MEDIA_ROOT" type="string">

  Example: `"/var/www/example.com/media/"`
</ParamField>

<ParamField path="MEDIA_URL" type="string">
  URL that handles media served from `MEDIA_ROOT`.
  
  Example: `"/media/"` or `"http://media.example.com/"`
</ParamField>

## Authentication

<ParamField path="AUTH_USER_MODEL" type="string">
  The model to use for user authentication.
  
  Default: `"auth.User"`
</ParamField>

<ParamField path="AUTHENTICATION_BACKENDS" type="list">
  List of authentication backend classes.
  
  Default: `["django.contrib.auth.backends.ModelBackend"]`
</ParamField>

<ParamField path="LOGIN_URL" type="string">
  URL where requests are redirected for login.
  
  Default: `"/accounts/login/"`
</ParamField>

<ParamField path="LOGIN_REDIRECT_URL" type="string">
  URL where requests are redirected after login.
  
  Default: `"/accounts/profile/"`
</ParamField>

<ParamField path="LOGOUT_REDIRECT_URL" type="string">
  URL where requests are redirected after logout.
  
  Default: `None`
</ParamField>

<ParamField path="PASSWORD_RESET_TIMEOUT" type="int">
  Number of seconds a password reset link is valid.
  
  Default: `259200` (3 days)
</ParamField>

<ParamField path="PASSWORD_HASHERS" type="list">
  List of password hasher classes. The first hasher is the preferred algorithm.
  
  Default:
  ```python
  [
      "django.contrib.auth.hashers.PBKDF2PasswordHasher",
      "django.contrib.auth.hashers.PBKDF2SHA1PasswordHasher",
      "django.contrib.auth.hashers.Argon2PasswordHasher",
      "django.contrib.auth.hashers.BCryptSHA256PasswordHasher",
      "django.contrib.auth.hashers.ScryptPasswordHasher",
  ]
  ```
</ParamField>

  Default: `False`
</ParamField>

<ParamField path="SESSION_COOKIE_HTTPONLY" type="bool">

  Default: `True`
</ParamField>

<ParamField path="SESSION_COOKIE_SAMESITE" type="string">
  Value of SameSite flag on session cookie.
  
  Default: `"Lax"`
</ParamField>
